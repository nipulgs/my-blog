# Install DSpace 6.3 on Ubuntu 20.04 LTS
This is an updated blog post over the installation of **DSpace 6.3 on Ubuntu 24.04 LTS**.  
In that guide, we learned how to install DSpace on Ubuntu 24.04 LTS.  
Here, we’ll install DSpace 6.3 on **Ubuntu 24.04 LTS**.  

---

## Hardware Recommendations

### Minimum DSpace Production
- 3–4 GB RAM  
- 20 GB Storage  

### Mid-range DSpace Production
- 5–6 GB RAM  
- 200 GB Storage  

### High-end DSpace Production
- 9–10 GB RAM  
- 1 TB Storage  

---

## Software Requirements
- Java JDK 8  
- Apache Maven  
- Apache Ant  
- PostgreSQL  
- Apache Tomcat  

---

### Create DSpace User

```bash
sudo useradd -m dspace
sudo passwd dspace
```
Install PostgreSQL
```bash
sudo apt install postgresql postgresql-client -y
```
Create a PostgreSQL user:
```bash
sudo su postgres
createuser -U postgres -d -A -P dspace
exit
```
Create a DSpace database:
```bash
sudo -u dspace createdb -U dspace -E UNICODE dspace
```
Activate the pgcrypto extension:
```bash
sudo su postgres
psql --username=postgres dspace -c "CREATE EXTENSION pgcrypto;"
exit
```
Edit PostgreSQL configuration:
```bash
sudo nano /etc/postgresql/12/main/pg_hba.conf
```
Add this line at the bottom:
```sql
local  all  dspace  md5
```
Restart PostgreSQL:
```bash
sudo systemctl restart postgresql
sudo systemctl status postgresql
```
### Building DSpace
Install OpenJDK 8:

```bash
sudo apt install openjdk-8-jdk -y
```
If you have another JDK installed, set Java 8 as default:

```bash
sudo update-alternatives --config java
```

Install Ant and Maven:

```bash
sudo apt install ant maven -y
```
Create required folders:

```bash
sudo mkdir /dspace

sudo chown dspace /dspace

sudo mkdir /build

sudo chmod -R 777 /build

cd /build
```
Download DSpace 6.3:

```bash
wget https://github.com/DSpace/DSpace/releases/download/dspace-6.3/dspace-6.3-src-release.tar.gz
```
Extract:

```bash
sudo tar xzvf dspace*.tar.gz
```
Move to source folder:

```bash
cd dspace-6.3-src-release
```
Copy local.cfg file:

```bash
sudo cp dspace/config/local.cfg.EXAMPLE dspace/config/local.cfg
```
Build DSpace:

```bash
sudo mvn -U package
```
Install DSpace:

```bash
cd dspace/target/dspace-installer
sudo ant fresh_install
```
###  Apache Tomcat Installation

Download and extract Apache Tomcat 9:

```bash
cd /opt
sudo wget https://downloads.apache.org/tomcat/tomcat-9/v9.0.41/bin/apache-tomcat-9.0.41.tar.gz
sudo tar xzvf apache-tomcat-9.0.41.tar.gz
```
Rename folder:

```bash
sudo mv apache-tomcat-9.0.41 tomcat
```
Edit /etc/profile:

```bash
sudo nano /etc/profile
```
Add these lines:

```bash
export JAVA_HOME=/usr/lib/jvm/java-8-openjdk-amd64
export CATALINA_HOME=/opt/tomcat
```
Copy DSpace webapps:

```bash
sudo cp -r /dspace/webapps/* /opt/tomcat/webapps
```
### Tomcat Service Script
Create init script:

```bash
sudo nano /etc/init.d/tomcat
```
Paste:

```bash
#!/bin/bash
### BEGIN INIT INFO
# Provides:        tomcat8
# Required-Start:  $network
# Required-Stop:   $network
# Default-Start:   2 3 4 5
# Default-Stop:    0 1 6
# Short-Description: Start/Stop Tomcat server
### END INIT INFO

PATH=/sbin:/bin:/usr/sbin:/usr/bin

start() {
 sh /opt/tomcat/bin/startup.sh
}

stop() {
 sh /opt/tomcat/bin/shutdown.sh
}

case $1 in
  start|stop) $1;;
  restart) stop; start;;
  *) echo "Run as $0 <start|stop|restart>"; exit 1;;
esac
```
Make executable and enable service:
```bash
sudo chmod +x /etc/init.d/tomcat
sudo update-rc.d tomcat defaults
```
Start Tomcat:

```bash
sudo service tomcat start
sudo service tomcat status
```
